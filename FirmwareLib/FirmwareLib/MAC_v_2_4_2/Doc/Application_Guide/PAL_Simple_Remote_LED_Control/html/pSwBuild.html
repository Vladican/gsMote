Compiling the Software

A build can be started either using the attached AVR Studio project file (aps-file) or on the command line.

The AVR Studio project file "Simple_Remote_LED_Control.aps" is located in directory Applications/PAL_Examples/Simple_Remote_LED_Control/ATMEGA128RFA1_RCB_6_3_PLAIN. The project can be opened simply by double clinking this aps file. Then the entire project can be rebuilt and started directly from AVR Studio.

The Makefile for buiiding the example application from the command line is located in the directory ATMEGA128RFA1_RCB_6_3_PLAIN/GCC. The build process can be started with the following command: 

  make -c ATMEGA128RFA1_RCB_6_3_PLAIN/GCC [CHAN=${chan}]

The resulting files for the standard build (w/o CHAN option) are: 

    ATMEGA128RFA1_RCB_6_3_PLAIN/GCC/simple_remote_led_control.elf ... object code elf file.
    ATMEGA128RFA1_RCB_6_3_PLAIN/GCC/simple_remote_led_control.hex ... object code hex file for programmer.
    ATMEGA128RFA1_RCB_6_3_PLAIN/GCC/simple_remote_led_control.lss ... listing file
    ATMEGA128RFA1_RCB_6_3_PLAIN/GCC/simple_remote_led_control.map ... symbol map file
    ATMEGA128RFA1_RCB_6_3_PLAIN/GCC/simple_remote_led_control.eep ... eeprom contents (unused)

In case that the channel parameter is changed via the the command line, e.g. with: 

  make -c ATMEGA128RFA1_RCB_6_3_PLAIN/GCC CHAN=17
the files are named simple_remote_led_control_ch${chan}, in this case simple_remote_led_control_ch17.*.
